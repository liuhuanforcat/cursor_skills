# My Skills

一个用于 [Cursor](https://cursor.com) 的 **Agent Skills（技能）** 仓库，用于存放可复用的技能定义，让 AI 助手在相关任务中自动遵循既定规范和最佳实践。

## 项目简介

本仓库以 Markdown 形式维护 Cursor 的 Agent Skills。每个技能对应一个 `SKILL.md` 文件，描述了在特定领域（如前端设计、代码规范等）中 AI 应遵循的步骤、规范和示例。当用户提出相关需求时，Cursor 会优先读取并应用对应技能，从而产出更专业、一致的结果。

适合用作个人或团队的知识库与「AI 协作规范」，便于在多项目中统一风格和质量。

## 目录结构

```
my-skills/
├── README.md                 # 本简介文档
└── .cursor/
    └── skills/               # 技能目录
        └── frontend-design/  # 前端设计（Ant Design）技能
            └── SKILL.md
```

## 已有技能

| 技能目录            | 说明                                                                                                                                    |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **frontend-design** | 创建符合 **Ant Design** 规范的企业级中后台界面。涵盖布局、组件、视觉与交互规范，以及排版与美学实践，适用于管理系统、后台 Admin 等场景。 |

更多技能可陆续在 `.cursor/skills/` 下按目录添加各自的 `SKILL.md`。

## 如何使用

1. **在 Cursor 中启用**  
   将本仓库克隆或链接到你的工作区，或把 `.cursor/skills/` 下的技能目录复制到你的项目的 `.cursor/skills/` 中。

2. **触发技能**  
   在与前端、中后台界面等相关的对话中，Cursor 会根据「可用技能」自动匹配并应用 `frontend-design` 等技能；你也可以在提问时明确提及「按 Ant Design 规范」或「用 frontend-design 技能」。

3. **扩展技能**  
   在 `.cursor/skills/` 下新建目录（如 `api-design`、`testing`），在其中添加 `SKILL.md`，并按 Cursor 技能格式编写说明与规范即可。

## 技能格式说明

每个技能的 `SKILL.md` 通常包含：

- **元数据**：`name`、`description` 等，便于识别与匹配。
- **设计思路 / 原则**：该领域需要先考虑的问题（如业务场景、信息架构）。
- **规范与约定**：具体的技术规范、组件用法、样式与交互标准。
- **示例与模板**：可复制的代码或结构示例（如需要）。

具体格式可参考 [Cursor 官方技能文档](https://docs.cursor.com) 或本仓库中的 `frontend-design/SKILL.md`。

## 许可与贡献

各技能文件顶部可注明版权与许可（如 `license`）；未单独声明的部分可视为随本仓库的默认许可。  
欢迎在遵循现有结构的前提下，新增或修改技能内容。
